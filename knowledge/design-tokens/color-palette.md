# Color Palette Design Tokens
 
## Overview
 
Our color palette is defined using a semantic approach, ensuring consistency and flexibility across all brand applications. Colors are categorized into brand, functional, and neutral palettes.
 
## Brand Colors
 
*   `--color-brand-primary`: Main brand color, used for primary actions and key branding elements.
*   `--color-brand-secondary`: Supporting brand color, used for secondary actions or complementary elements.
 
## Functional Colors
 
*   `--color-functional-success`: Used for positive actions or success messages (e.g., green).
*   `--color-functional-warning`: Used for cautionary messages or warnings (e.g., orange).
*   `--color-functional-error`: Used for error states or critical information (e.g., red).
*   `--color-functional-info`: Used for informational messages (e.g., blue).
 
## Neutral Colors
 
*   `--color-neutral-text-default`: Default text color.
*   `--color-neutral-text-light`: Lighter text color for secondary information.
*   `--color-neutral-background-default`: Default page background color.
*   `--color-neutral-border`: Default border color for UI elements.
 
## Usage
 
Always reference colors via their semantic token names (e.g., `var(--color-brand-primary)`) rather than hardcoded hex values.
